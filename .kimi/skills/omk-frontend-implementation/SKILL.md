## Rules

1. Read DESIGN.md before writing UI code.
2. Reuse existing components and tokens.
3. Do not invent arbitrary colors or spacing.
4. Validate responsive behavior.
5. Check accessibility (a11y) basics.
6. Run lint, typecheck, and build before claiming done.
